# Publishing Notes

`ARTICLE.md` uses absolute GitHub-hosted image URLs so the images render in GitHub's Markdown preview.

When publishing to X Articles, do not rely on Markdown image syntax or repo-relative paths. Use the X Article image controls and upload the images manually in this order:

1. `assets/article-header.png`
2. `assets/screenshot-comparison.png`
3. `assets/quantitative-table.png`
4. `assets/judge-scores.png`

Then remove any raw Markdown image syntax if the X composer leaves it as text.
